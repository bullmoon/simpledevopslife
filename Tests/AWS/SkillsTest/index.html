Amazon: "Back up your on-premises applications to the cloud using AWS Storage Gateway"
Customers use Storage Gateway to protect their on-premises applications and to reduce backup infrastructure and administration costs. They use Storage Gateway to back up files, applications, databases, and volumes to Amazon S3, Amazon S3 Glacier, Amazon S3 Glacier Deep Archive, and Amazon EBS – through files, volumes, snapshots, and virtual tapes in AWS. Some customers use Storage Gateway to seamlessly complement their existing storage infrastructure to offload and/or expand their on-premises storage capacity. Since there is no hardware procurement needed, these deployments are faster and shorter.

What is the best practice for maintaining Windows EC2 instances and applying updates?
- Use AWS System Manager Path Manager to find and patch instances that require updates during a set maintenance window.
- Create maintenance schedule that an employee must fill out each week confirming a visual inspection of each instance was conducted and which patches were applied.
- Install Windows Server Update Services on your primary Active Directory controller.
- Turn on auto Update in Windows Update on each EC2 that is launched, or create your own AMI with this feature enabled and launch all of your EC2 instances from this AMI.
Amazon: Patch Manager, a capability of AWS Systems Manager, automates the process of patching managed nodes with both security related and other types of updates. You can use Patch Manager to apply patches for both operating systems and applications. (On Windows Server, application support is limited to updates for applications released by Microsoft.) You can use Patch Manager to install Service Packs on Windows nodes and perform minor version upgrades on Linux nodes.
